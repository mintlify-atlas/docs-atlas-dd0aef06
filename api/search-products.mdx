## Overview

The `search_products` tool searches the WalletWake product catalog for items matching a query. It returns product names, prices, and currencies. This allows the agent to look up actual product prices when users mention items they want to buy.

**Tool name:** `search_products`

## When to use this tool

Use this tool when:
- A user mentions a product and you need to look up its actual price
- A user asks "how much is [product]?"
- You need real price data before calling `calculate_real_cost`
- A user wants to browse available products in a category

## Input schema

<ParamField path="query" type="string" required>
  Search query for the product catalog. Can be a product name, category keyword, or the special value "all" to return all products.
  
  Examples:
  - `"headphones"` - searches for headphones
  - `"macbook"` - searches for MacBook products  
  - `"gucci"` - searches for Gucci items
  - `"all"` - returns entire catalog
  - `""` (empty string) - also returns entire catalog
</ParamField>

### Example input

```json
{
  "query": "headphones"
}
```

## Output schema

<ResponseField name="results" type="array">
  Array of matching products.
  
  <Expandable title="Product object structure">
    <ResponseField name="name" type="string">
      The full product name/title.
    </ResponseField>
    
    <ResponseField name="price" type="string">
      The product price as a decimal string (e.g., "348.00").
    </ResponseField>
    
    <ResponseField name="currency" type="string">
      The currency code (e.g., "USD").
    </ResponseField>
  </Expandable>
</ResponseField>

## Search behavior

The search performs case-insensitive substring matching against product titles:

- **Query:** `"sony"` → Returns Sony headphones and PlayStation 5
- **Query:** `"guitar"` → Returns Fender Stratocaster
- **Query:** `"all"` or `""` → Returns all 10 products
- **Query:** `"nintendo"` → Returns empty array (no matches)

## Complete conversation example

**User:** How much are the Sony headphones?

**Agent workflow:**
1. Calls `search_products` with query "sony headphones"
2. Receives product results with price $348.00
3. Responds: "The Sony WH-1000XM5 Wireless Noise Canceling Headphones are $348. Want me to show you what that really means?"
4. If user says yes, calls `calculate_real_cost` with price 348
5. Renders `spending-insight` component with the data

## Chaining with other tools

Common workflow pattern:

```
User mentions product
    ↓
[search_products] - Look up actual price
    ↓
[calculate_real_cost] - Convert to equivalents
    ↓
Render spending-insight component
```

**Example:**

**User:** "I'm thinking about getting a PS5"

1. **Search:** `search_products({query: "ps5"})`
   - Returns: PlayStation 5 at $499.00

2. **Calculate:** `calculate_real_cost({price: 499, productName: "PlayStation 5"})`
   - Returns: equivalents, monthly impact, investment potential

3. **Render:** Display `spending-insight` component with all the data
