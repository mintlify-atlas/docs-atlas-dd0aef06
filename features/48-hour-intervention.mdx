The 48-hour intervention is WalletWake's most powerful feature. When you trigger a funeral for an impulse purchase, you can't access normal app functionality for exactly 48 hours. This forced pause leverages behavioral psychology to break the impulse-to-purchase loop.

## Why 48 hours?

Research shows that the emotional intensity of impulse desire decreases significantly after 24-48 hours. By enforcing this waiting period, WalletWake gives you time to:

- Process the initial dopamine rush
- Evaluate whether you actually need the item
- Consider opportunity costs
- Explore alternatives
- Make a rational decision instead of an emotional one

<Info>
  The "sleep on it" principle, extended to two full days.
</Info>

## How the timer works

### Timestamp-based calculation

When a funeral starts, the system records the exact timestamp:

```typescript
const newState: FuneralState = {
  isActive: true,
  productTitle: product.title,
  productPrice: product.price.value,
  currency: product.price.currency,
  imageUrl: product.image.imageUrl,
  deathDate: Date.now()  // Current timestamp in milliseconds
};
```

### Resurrection calculation

The resurrection time is calculated by adding 48 hours in milliseconds:

```typescript
const resurrectionTime = state.deathDate + (48 * 60 * 60 * 1000);
// 48 hours × 60 minutes × 60 seconds × 1000 milliseconds
```

### Countdown display

The timer updates every second, showing the remaining time:

```typescript
const timer = setInterval(() => {
  const target = state.deathDate + 48 * 60 * 60 * 1000;
  const diff = target - Date.now();

  if (diff <= 0) {
    setTimeLeft("00:00:00");
    clearInterval(timer);
    return;
  }

  const h = Math.floor(diff / (1000 * 60 * 60));
  const m = Math.floor((diff % (1000 * 60 * 60)) / (1000 * 60));
  const s = Math.floor((diff % (1000 * 60)) / 1000);

  setTimeLeft(
    `${h.toString().padStart(2, "0")}:${m.toString().padStart(2, "0")}:${s.toString().padStart(2, "0")}`
  );
}, 1000);
```

<Note>
  The countdown displays in monospaced font with tabular numbers for consistent width and readability.
</Note>

## State persistence

The intervention survives across sessions by storing state in localStorage:

### Saving state

```typescript
localStorage.setItem("walletwake_funeral", JSON.stringify(newState));
```

### Loading state

```typescript
useEffect(() => {
  const saved = localStorage.getItem("walletwake_funeral");
  if (saved) {
    try {
      setState(JSON.parse(saved));
    } catch (e) {
      console.error("Failed to parse funeral state");
    }
  }
  setIsLoaded(true);
}, []);
```

## Resurrection check

The `useImpulseStore` hook exports a boolean that indicates whether the 48 hours have passed:

```typescript
const isResurrected = () => {
  if (!state) return false;
  const now = Date.now();
  const resurrectionTime = state.deathDate + (48 * 60 * 60 * 1000);
  return now > resurrectionTime;
};

return {
  funeralState: state,
  startFuneral,
  clearFuneral,
  isResurrected: isResurrected(),  // Exported as boolean
  isLoaded
};
```

When `isResurrected` becomes `true`:
- The funeral view is hidden
- Normal app functionality returns
- The state can be cleared

<Tip>
  The app doesn't automatically clear the state after resurrection—it waits for user action to preserve the data for potential future reference.
</Tip>

## Developer fast-forward

For testing purposes, the hook includes a `fastForward()` function:

```typescript
fastForward: () => {
  if (!state) return;
  const oldState = {
    ...state,
    deathDate: Date.now() - (49 * 60 * 60 * 1000)  // Set to 49 hours ago
  };
  setState(oldState);
  localStorage.setItem("walletwake_funeral", JSON.stringify(oldState));
  window.location.reload();
}
```

This sets the death date to 49 hours in the past, instantly ending the funeral and triggering resurrection.

<Warning>
  This function is only for development and demo purposes. It reloads the entire page after updating the state.
</Warning>

## Behavioral psychology principles

The 48-hour intervention leverages several psychological concepts:

### Temporal discounting

People value immediate rewards more than future ones. By forcing a delay, the intervention reduces the perceived value of the impulse purchase.

### Cognitive reappraisal

The funeral view reframes the purchase as a "death" rather than an acquisition, triggering emotional processing that leads to rational reconsideration.

### Choice architecture

The intervention makes the impulsive choice (buying) harder and the rational choice (waiting) the default path.

### Precommitment

By triggering the funeral, you're precommitting to a 48-hour wait, removing the burden of willpower from moment-to-moment decisions.

<Info>
  These principles are foundational to behavioral economics and have been validated in numerous studies on self-control and decision-making.
</Info>

## What happens during the 48 hours

While the funeral is active:

1. **Full-screen takeover** — The funeral view replaces normal navigation
2. **Reality checks** — Price conversions show what you could buy instead
3. **Alternatives** — Cheaper options are displayed if available
4. **Countdown pressure** — The ticking timer creates awareness of time passing
5. **Shameful escape** — The "buy anyway" button requires acknowledging lack of self-control

## After 48 hours: the clarity screen

When the countdown reaches zero, the funeral view is replaced by a **clarity screen** that asks you to make a final decision with a clear mind.

### What you see

The clarity screen displays:

- **Success indicator** — A green checkmark in a pulsing circle, acknowledging that you've successfully waited
- **Message** — "The impulse has passed." Reinforcing that you've completed the intervention
- **Reflection prompt** — "You waited 48 hours. If you still want this, it's a deliberate choice—not a reflex."
- **Product recap** — Small card showing the product image, title, and price
- **Two action buttons:**
  - **"Purchase with clarity"** — Opens the product page (typically on eBay) in a new tab
  - **"I don't want it anymore"** — Dismisses the funeral and clears the state

<Info>
  The clarity screen validates your patience while making both choices equally accessible—no shame attached to either decision.
</Info>

### Making your choice

#### If you decide to buy

Clicking **"Purchase with clarity"** will:
1. Open the product URL in a new browser tab
2. Clear the funeral state from localStorage
3. Return you to the normal app interface

This acknowledges that after 48 hours of reflection, your purchase is intentional.

#### If you decide not to buy

Clicking **"I don't want it anymore"** will:
1. Clear the funeral state immediately
2. Return you to the chat interface
3. Free you to continue using the app normally

Most users choose this option—the intervention works.

<Tip>
  The clarity screen doesn't automatically redirect or pre-select an action. Both choices are neutral, respecting your autonomy after the intervention period.
</Tip>

## Clearing the funeral

The `clearFuneral()` function removes the state:

```typescript
const clearFuneral = () => {
  setState(null);
  localStorage.removeItem("walletwake_funeral");
};
```

This is called when:
- You click "I don't want it anymore" on the clarity screen
- You complete a purchase via "Purchase with clarity"
- You start a new funeral for a different product
- During development/testing
